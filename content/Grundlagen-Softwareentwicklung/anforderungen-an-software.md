# Funktionale Anforderungen
Die funktionalen Anforderungen beschreiben, was ein System tun soll.
Sie legen fest, welche Funktionen, Prozesse udn Abläufe des System unterstützen oder bereitstellen muss, um die fachlichen Bedürfnisse der Anwender zu erfüllen.
Sie beziehen sich direkt auf das Verhalten des Systems und sind in der Regel konkret überprüfbar (z.B. duch Tests oder Use Cases).

## Beispiel
- Das System soll Benutzeranmeldungen verabeiten können.
- Das System soll Bestellungen erfassen, speichern und bearbeiten können.
- Der Benutzer soll Passwörter zurückstezen können.

---

# Nicht Funktionale Anforderungen
Die nicht-funktionalen Anforderungen beschreiben, wie gut ein System seine Funktionen ausführt.
Sie betreffen Qualitätsmerkmale wie Leistung, Sicherheit, Benutzerfreundlichkeit, Zuverfässigkeit oder Wartbarkeit.
Diese Anforderungen definieren also Rahmenbedingungen und Qualitätsstandards, die das System erfüllen muss, um akzeptabel und eefizient nutzbar zu sein.

## Beispiel
- Die Anmeldung soll innerhalb von 3 Sekundne erfolgen.
- Die Anmeldung soll DSGVO konform sein.
- Die Anwendung soll auch bei 10.000 Anfragen gleichzeit stabil bleiben.
- Die Benutzeroberfläche soll intuitiv und barrierefrei gestaltet sein.
